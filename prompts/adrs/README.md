# ADRs (Architecture Decision Records)

## Objetivo

Registrar decisões estruturais de arquitetura/produto que precisam permanecer rastreáveis ao longo do tempo.

Os ADRs aqui complementam os planos:
- `prompts/master-tecnico.md`
- `prompts/master-produto-snap.md`
- `prompts/entregas-api-snap-v2.md`

## Convenção

- Numeração crescente: `0001-...md`, `0002-...md`
- Não editar a decisão passada sem registrar mudança de status/contexto
- Se a decisão mudar:
  - criar novo ADR
  - referenciar o anterior

## Estrutura sugerida

- Status
- Contexto
- Decisão
- Consequências
- Relação com planos
